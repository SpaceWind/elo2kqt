#This file will describe new map format;

FIRST:
You should describe scene:
Scene consist of:

- maps
- sprite resources
- sound resources
- config resources

Before map starts it will load all needed resources for the map.
Then first map in the scene will be loaded.

map can output some data internal (available in this scene only and removed after scene is done)
or external (available throught the game).

@scene
	resources = {%soundName% %filename% %loop%} as sounds
	resources = {%soundName% %filename% %loop%} as sounds
	resources = {%filename%} as config
	resources = {%spriteType%} as sprites
	maps = {map1 map2 map3}
	name = {name}
/scene


Сцена компилируется в байт код и интерпритируется в режиме выполнения.
Сама сцена может использовать как звуки и спрайты из стандартного набора, так и содержать внутри себя.

Структура .scene файла:

MAGIC 				: 4 bytes
POINTER to scene config 	: 4 bytes
POINTER to map config start	: 4 bytes
POINTER to sprite descriptors	: 4 bytes
POINTER to sprite resources	: 4 bytes
POINTER to sound resources	: 4 bytes
POINTER to config resources	: 4 bytes


Каждый блок данных в бинарном формате сцены состоит из:

TYPE BLOCK MAGIC		: 4 bytes
SIZE in bytes			: 4 bytes
qCompressed DATA		: 4 bytes
POINTER next			: 4 bytes


Что такое CONFIG:
Задает внутренние переменные для сцены. Может быть импортировано из текущих переменных.

@config
IMPORT CURRENT	//копирует все текущие глобальные параметры в текущие для сцены
IMPORT DEFAULT	//копирует все глобальные параметры по умолчанию в текущие для сцены
EXPORTMAP AS CURRENT //копирует все параметры карты в глобальные текущие переменные
EXPORTSCENE AS DEFAULT	//копирует все параметры сцены в глобальные параметры по умолчанию 
SET a 50
ADD b 30
(технически 'a' = SceneName::a)
/config


Изменения в переменных, три уровня переменных:
MAP SET
SCENE SET	
GLOBAL SET

Различаются тем, где они становятся видимыми.

При выгрузке карты все переменные данной карты уничтожаются.
При выгрузке сцены все переменные данной сцены уничтожаются.
Переменные GLOBAL не уничтожаются вплоть до окончания работы программы.
